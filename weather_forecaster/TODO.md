# Implementation Checklist

This checklist ensures all steps are followed for future implementations.

## Steps to Follow

- [x] Read project rules file (etl-rules.md) before implementing
- [ ] Check architecture.md for system design questions
- [ ] Check api-reference.md for how to use sources
- [ ] Check data-dictionary.md for type mappings
- [ ] Check deployment.md for Docker/CI-CD info
- [ ] Check unit_testing_guide.md for writing tests
- [ ] Implement the feature/function
- [ ] Create unit tests following testing guidelines
- [ ] Verify tests pass with pytest
- [ ] Update documentation if needed

## Usage

When requesting new features, reference this checklist to ensure all steps are completed.
